## How to Use This Plan

1. **Follow the order**: Complete each day's problems. Read the solution first, understand the approach, then write the code yourself.
2. **Three passes per problem**: 1st pass — read and understand. 2nd pass — write from scratch. 3rd pass — write with a timer.
3. **Take notes**: Record your pitfalls and insights for each problem.
4. **Weekly review**: Every Sunday, revisit all problems from the past week. Redo the ones you got wrong.
5. **📋 Track progress**: Use [`progress-tracker.md`](../progress-tracker.md) to log daily completion and mastery level.

## Study Tips

### The Three-Pass Method

1. **First pass (understand)**: Read the problem → read the solution → understand the approach → type out the code
2. **Second pass (write from memory)**: Without looking at the solution, write the complete code → pass tests
3. **Third pass (timed)**: Complete within interview time limits, bug-free

### If You Get Stuck

1. Think for 10-15 minutes. At minimum, come up with a brute-force approach.
2. If completely stuck, read the "approach" section of the solution — don't look at the code.
3. After understanding the approach, try writing the code yourself.
4. If you still can't write it, read the full solution code, then redo the problem the next day.

### Language

Solutions use **Python** (most readable and interview-friendly). If you use Java/C++/JavaScript, the logic is identical — just translate the syntax.
